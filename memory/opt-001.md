## 教训
- **测试设计要点**: health-check.sh 探测真实外部服务(端口/进程/外部配置文件),
  无法 mock 整个环境。所以测试不该假设环境"健康"或"异常", 而应验证
  **退出码与标记文件的一致性契约**——这个契约与环境状态无关, 是稳定可测的。
- **隔离测试**: 用 `sed` 复制 health-check.sh 并替换命令为不存在的命令, 构造必失败副本;
  用 `mktemp -d` 建临时 state 目录隔离标记文件, 避免污染真实 state/。
- **CLAUDE.md 边界**: health-check.sh 第 4/5 项只读访问 `/home/opc_uname/.openclaw/openclaw.json`
  和 `/home/opc_uname/.agents/skills/`。这些是项目目录外路径, 但只是**只读探测**,
  不违反"不修改"边界。后续优化若想动这些检测项要谨慎。
- **snapshot.sh 行为**: 每次跑都会 `git add -A` + 自动 commit + 写 state/stable-tags.json。
  所以 stable-tags.json 会持续增长, 不在 .gitignore 里(预期入库)。注意不要让它膨胀失控。
- **snapshot.sh 同名 tag**: 若 tag 已存在, `git tag` 在 `set -e` 下直接退出无提示。非 bug, 是设计选择。
